vitea

AI coding agent を並列運用するためのネイティブ macOS ターミナル。worktree とセッションを第一級オブジェクトとして扱う。

cmux の UI 品質 × ccmanager の worktree 管理 × 1 worktree : N セッション

Screen 01

メイン画面

1
リポジトリ行。複数リポジトリ(複数ディレクトリ)を登録して1つのサイドバーで管理。折りたたんでも waiting 数のバッジは集約表示される(最下段の infra 参照)。
2
worktree 行。ブランチ名、親ブランチとの ahead/behind(↑3 ↓1)、staged(+ 緑)/unstaged(! オレンジ)の有無を常時表示。変更が無ければ clean。
3
セッション行。同一 worktree 配下に claude #1 / claude #2 / codex #1 と複数セッションをぶら下げられる(核心要件)。⌘1〜9 で直接ジャンプ。
4
libghostty サーフェス。フル機能のターミナル。~/.config/ghostty/config のテーマ・フォントを継承。非表示のセッションもバックグラウンドで生存。
5
ステータスバー。リポジトリ横断で全セッションの状態を集計。⌘⇧U で最新の入力待ちへジャンプ(別リポジトリでも飛べる)。

Spec

状態インジケータとキーマップ

状態検出は OSC 通知シーケンスを優先し、画面テキストのパターンマッチ + idle デバウンス(1.5s)をフォールバックにする二段構え。

Session State

busy — エージェントが処理中。通知しない。
waiting — 入力待ち。バッジ + macOS 通知 + フック発火。
idle — プロンプト待機。1.5s のデバウンス後に確定。

Keymap
